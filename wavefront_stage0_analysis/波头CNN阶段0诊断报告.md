# 波头 CNN 阶段 0 诊断报告

生成对象：Colab 导出的 `wavefront_cnn_colab_results.zip`  
测试集：hard 伪标签，543 个相级样本，214 个录波事件  
判定：**阶段 0 自动量化已完成；标签主因只能得到关联证据，仍需 Gold 人工复核作因果确认。**

## 1. 数据与计算校验

- 导出清单中的 8 个结果文件 SHA256 全部通过。
- `test_predictions.csv` 含 543 条唯一预测，全部一对一关联到 `phase_labels.csv`。
- 重新计算 MAE、RMSE、P95 与 `test_metrics.json` 的绝对差分别为 `1.3657e-05`、`3.29025e-05`、`0.00012207` samples。
- 从 `best_finetune.pt` 重新推理，与导出预测最大差 `0.000488` samples，检查点与 CSV 一致。

## 2. 粗 bin 与 offset 误差分解

| 指标 | 结果 |
|---|---:|
| 粗 bin 宽度 | 32 samples |
| 粗 bin Top-1 | 58.01% |
| 粗 bin Top-3 | 82.14% |
| 粗 bin Top-5 | 87.11% |
| 粗 bin 选错数量 | 228 / 543 |
| 选对 bin 后 offset MAE | 2.528 samples |
| 选对 bin 后 offset P95 | 8.680 samples |
| 选错 bin 后整体 MAE | 1544.287 samples |
| 选错 bin 后整体 P95 | 3869.328 samples |

结论：灾难误差的直接机制是粗候选/bin 选择错误。offset 头在粗 bin 正确时的误差明显较小，因此下一阶段应优先解决候选召回和首波排序，而不是继续单独增加 offset 回归容量。

## 3. 现有标签风险标记分层

这里的“未标记”只表示没有命中现有的跨相 spread 和局部导数规则，不代表人工确认正确。

| 分组 | 相级样本 | 事件 | MAE samples | P95 samples | >64 samples |
|---|---:|---:|---:|---:|---:|
| 未命中现有风险 | 436 | 180 | 483.54 | 3111.26 | 22.71% |
| 命中任一风险 | 107 | 46 | 1327.75 | 4140.20 | 65.42% |

风险组减未标记组的 MAE 差为 `844.21` samples；按录波事件聚类 bootstrap 的 95% CI 为 `[473.00, 1224.79]`。

解释边界：如果区间不跨 0，只能说明现有风险标记与模型误差存在统计关联；风险标记和测试真值均来自同一自动标注流程，不能据此单独证明标签错误造成全部误差。

## 4. 最差 30 例

- 最差 30 例中 `15/30` 命中已知风险，测试全集风险占比为 `19.71%`。
- 单侧 Fisher 检验：odds ratio=`4.576`，p=`0.000119496`。
- 该结果用于确定 Gold 复核优先级；由于样本按模型误差选取，不能作为无偏的标签错误率估计。

## 5. 阶段 0 Go/No-Go

**Go：进入数据集 v2、Gold 标注和候选排序原型。**

理由：

1. 模型检查点和逐样本结果完整，阶段 0 数值可复现。
2. 粗 bin 选错与大误差直接对应，证明“候选选择”是当前模型的直接失败环节。
3. 现有标签风险规则可用于提高人工复核队列效率，但不能替代 Gold 真值。
4. 即使 offset 表现较好，标签依赖裁剪仍是部署阻断问题，因此数据集 v2 不能跳过。

## 6. 知识梳理与工程边界

数据流：`Colab 预测 → 标签/录波关联 → 风险分层 → 检查点重放 → bin/offset 分解 → Gold 复核队列`。

工程上必须把三类指标分开：候选生成 recall、候选排序 Top-1、选对候选后的 offset 误差。当前结果只能评价伪标签坐标拟合，不能直接证明真实双端测距精度。下一步应由 Gold 标注程序对随机分层样本和最差样本进行盲态复核，并用已知故障位置或可信双端事件完成外部验证。
