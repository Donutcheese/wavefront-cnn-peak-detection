# 波头 CNN 训练结果与根因分析

生成日期：2026-07-15  
对应 Notebook：`machine_learning/wavefront_cnn/wavefront_cnn_colab_all_in_one.ipynb`  
原始工程目录：`FaultLocation_demo`

## 1. 总体结论

当前这版波头 CNN 训练结果不适合作为工程可部署模型，也不适合作为后续单纯调参优化的基线。核心问题不是“模型容量不够”或“训练轮数不够”，而是监督目标和工程任务之间存在结构性错位：

1. 测试集使用的硬标签本身包含大量伪标签误差，模型评价在相当程度上是在衡量“是否模仿当前自动标注器”，而不是“是否找到了真实初始行波波头”。
2. 数据集裁剪依赖标签位置，形成了目标泄漏式的中心位置先验；真实上线推理时无法预先知道波头标签，因此这种训练分布不可复现。
3. 波形中常见多次突变、反射波、后续大幅值波头，当前模型用全局最大置信度解码，缺少“最早可信波头”的物理约束，容易从初始波头跳到后续强脉冲。
4. 三相标签没有被建模为同一个故障事件的物理一致性问题，而是按相独立学习，导致跨相冲突、缺相裁剪、伪标签漂移直接进入训练和测试。

因此，优化方向应从“继续调 CNN 参数”转为“重构数据集标签体系 + 改为候选波头排序/判别 + 引入三相和双端物理约束”。如果继续在当前伪标签和裁剪机制上训练，指标可能局部变好，但工程可信度不会同步提升。

## 3. 结果应该如何解读

从机器学习角度看，overfit sanity 能做到 MAE 约 0.46 samples，说明模型结构和训练代码本身没有明显“学不动”的问题。也就是说，模型不是因为太小、优化器完全失效、GPU 运行错误、loss 写错这类基础问题而失败。

真正的问题出现在泛化测试：测试 MAE 约 649.90 samples，而只使用位置先验的 baseline MAE 约 551.37 samples。模型甚至没有稳定超过一个非常弱的“标签位置常数预测”基线。这说明输入波形中的有效判别信息没有被当前训练目标稳定利用，或者标签噪声和任务定义让模型无法学习到一致规律。

从电气工程角度看，这个误差量级已经远超行波测距允许范围。若双端测距采用

```text
x = (L + v * Δt) / 2
```

单端波头时间误差约 519.92 us，在传播速度近似 `299.79 km/ms` 时，会带来约 `77.9 km` 的双端位置误差量级；P95 误差约 2650.94 us，则可能对应约 `397 km` 的位置误差量级。这个量级在输电线路故障定位中不可接受。

但还要注意：测试指标并不能直接等价为“模型比真实物理更差”。因为测试 hard label 本身来自自动标注器，不是真实人工金标准。部分最差样本中，模型预测到的早期突变反而可能比伪标签更符合“初始波头”语义，只是被错误伪标签惩罚了。

## 4. 根因分析

### 4.1 伪标签质量不足是最高优先级问题

数据集审计已经给出 `label_quality_passed=false`。关键风险包括：

| 证据 | 影响 |
|---|---|
| 168 个 hard labels 在 common crop 后丢失 | 说明裁剪和标签体系不稳定 |
| 226 / 1529 个文件跨相 hard 标签 spread > 64 samples | 三相同一故障事件的波头时间不一致 |
| 396 / 4592 个 hard labels 导数证据弱 | 标签可能落在平滑段或非真实突变处 |
| hard label 检测器 spread 中位数约 6.4 us，P95 约 30.4 us | 标签自身不确定性已经大于 Acc@4 这类严格指标 |
| 约 80.1% hard 标签检测器 spread 超过 4 samples | 用 ±4 samples 评价伪标签精度并不合理 |

自动标注器中，DWT、DS_SWT、BDWT 的一致性被用来提升 hard label 置信度，但 DWT 与 DS_SWT 都依赖 `db4` 小波，二者并非完全独立证据。多个相关检测器投票会高估标签置信度。

源码证据位置：

- `FaultLocation_demo/ningxia_python_project/net_faultlocation/dataset/wavefront_labeler.py`
- `FaultLocation_demo/ningxia_python_project/net_faultlocation/dataset/builder.py`
- `data/derived/wavefront_dataset_v1/audit_report.json`（本地审计输出，public repo 默认不提交）

### 4.2 当前裁剪方式引入了标签位置先验

数据集中约 97% 的 hard 标签落在窗口 30% 到 55% 的中心区域。这不是自然故障波形分布，而是 label-dependent common crop 造成的结果。换句话说，先用自动标签找到波头，再围绕标签裁剪窗口，然后再让模型预测这个标签。

这会造成两个问题：

1. 训练和测试都带有“标签应该在窗口中部”的强先验，导致 position-only baseline 已经很强。
2. 工程上线时没有真实标签，无法用同样方式裁剪，因此训练输入分布和部署输入分布不一致。

当前训练里的随机平移增强只有约 ±192 samples，占 8192 点窗口的 2.3%左右，无法消除这种中心位置先验。

这意味着当前数据集更像“已知波头附近的小窗口精定位任务”，而不是“从原始录波中自动找到初始行波波头”的工程任务。

### 4.3 多峰波形下，模型缺少“最早可信波头”约束

最差样本可视化显示，很多波形存在多个突变、反射波或后续更强的脉冲。当前 coarse-to-fine 模型本质上在 256 个粗 bin 中做全局最大响应选择，再输出 offset。这个结构适合“只有一个目标峰”的定位问题，但不适合“必须选最早真实波头，且后面可能有更强反射”的故障行波问题。

典型现象：

| 样本 | 现象 | 说明 |
|---|---|---|
| `062dd39bfc391c8fe60a1a0d:A` | A 相 hard label 在窗口后部，B/C 相在早期；模型预测接近早期事件 | 测试误差很大，但模型可能选中了更物理一致的事件 |
| `7264eba815c755ec0e09810f:A` | A 相 hard label 极靠后，B 相 hard label 被裁掉，C 相 soft label 在中部 | 标签体系本身不稳定 |
| `b587138331c8611a56dba803:C` | 三相 hard label 一致，但模型跳到后续相似强突变 | 模型缺少“首波优先”机制 |
| `4914dc29704ef1ad93afde18:C`、`00a7cef5313918e7ae828c7e:C` | 多个候选波头相似，预测跨 bin 跳变 | 单一全局 argmax 容易选错候选 |

因此，当前模型的主要失败模式不是 offset 回归不准，而是候选事件选择错了。一旦粗 bin 选错，误差会立刻达到几千 samples。

### 4.4 三相和双端物理约束没有进入监督闭环

当前训练样本按相独立学习，模型看到的是单相波形和单相标签。实际故障定位中，三相波头应当服从同一故障事件的时序结构，双端波头还要满足线路长度、传播速度和端点方向约束。

当前数据集中有 175 个 double-end pairs，其中物理有效 90 个，保守位置监督 57 个。也就是说，真正能支持双端测距监督的样本很少；而大部分相级标签并没有经过“能否产生合理双端测距”的物理反校验。

这会导致一个危险结果：模型在相级伪标签上看似训练，但最终双端测距时可能严重违反线路物理边界。

### 4.5 幅值分布重尾和归一化方式会放大训练不稳定

HDF5 中经过 baseline MAD 归一化后，部分样本仍存在极大幅值：

| 指标 | max_abs |
|---|---:|
| median | 113.36 |
| P75 | 498.77 |
| P90 | 1603 |
| P95 | 6335.8 |
| P99 | 10204 |
| max | 23368 |

约 22% hard 信号 `max_abs > 500`，约 14.1% `max_abs > 1000`。如果没有裁剪、`log1p` 压缩或更稳健的归一化，BatchNorm 很容易被大幅值样本主导。当前训练噪声增强约 0.025，相对这种重尾幅值几乎不起作用。

## 5. 反推优化方案

### 5.1 第一优先级：重建数据集 v2，而不是继续调参

建议把标签分成三层：

| 标签层级 | 来源 | 用途 |
|---|---|---|
| gold | 人工复核 + 双端物理一致性校验 | 作为验证集、测试集和模型选择依据 |
| silver | 严格自动标注，必须通过跨检测器、跨相、物理边界检查 | 可用于主训练，但需要权重和不确定性 |
| review / unlabeled | 冲突大、弱突变、多峰歧义、裁剪丢失 | 不作为硬监督，可用于候选生成、无监督预训练或人工复核队列 |

不建议再把当前 4592 个 hard label 全部等价当作强监督真值。当前数据可以保留，但要重新决定哪些样本可以承担监督责任。

### 5.2 第二优先级：取消标签依赖裁剪

训练和推理必须使用同一种可复现输入定义。推荐两种方式：

1. 基于 `.all` 文件头、触发时间、固定物理搜索窗构造输入，不使用标签决定窗口中心。
2. 使用滑窗或候选点局部窗口，让模型只判断局部候选是否为真实初始波头。

这样可以消除“标签在窗口中心”的人工先验，使模型真正学习波形特征，而不是学习裁剪流程留下的位置偏置。

### 5.3 第三优先级：从全局坐标回归改成候选波头排序

更适合工程的结构是“信号处理候选 + CNN 判别/排序”，而不是从 8192 点窗口中直接全局 argmax。

建议流程：

```mermaid
flowchart LR
  A["原始 .all/.vall 录波"] --> B["固定物理搜索窗"]
  B --> C["DS / MODWT / 变点检测生成 Top-K 候选"]
  C --> D["候选局部窗口: ABC + Clarke + 导数 + 小波能量"]
  D --> E["CNN/Transformer 候选判别"]
  E --> F["first-wave ranking + offset + uncertainty"]
  F --> G["三相一致性检查"]
  G --> H["双端线路物理约束"]
  H --> I["最终波头与故障距离"]
```

模型输出建议包括：

| 输出 | 含义 |
|---|---|
| `p_first` | 当前候选是否为初始真实波头 |
| `offset` | 候选附近的细粒度时间修正 |
| `uncertainty` | 不确定性，用于拒识和人工复核 |
| `phase_consistency` | 三相候选是否属于同一事件 |

训练损失不应只做单点坐标误差，还应加入排序约束：真实初始波头候选的分数必须高于后续反射波、后续大幅值波、噪声尖峰和慢变漂移。

### 5.4 第四优先级：加入电气物理约束

工程上需要明确以下约束：

1. 三相同一故障事件的首波应在合理时间容差内出现，不能随意相差几百到几千 samples。
2. 双端时间差必须能落入线路长度边界 `[0, L]`。
3. A/B 端点方向必须和线路拓扑一致，否则距离公式会出现系统性偏移。
4. 625 kHz 样本上采样到 1.25 MHz 不能凭空增加原始信息量，评价时应按原始采样率和统一采样率分别报告。
5. 对不满足物理约束的样本，不应强行给 hard label，而应转入 review 或弱监督。

### 5.5 第五优先级：稳健预处理和模型训练

建议修改点：

| 模块 | 优化 |
|---|---|
| 归一化 | 使用固定故障前窗口估计噪声，避免标签依赖 |
| 幅值处理 | signed `log1p`、robust clipping、保留原始幅值尺度作为 metadata |
| Norm 层 | 小 batch 或重尾幅值下优先试 GroupNorm / LayerNorm |
| 数据增强 | 加入后续强脉冲、反射波、噪声尖峰、慢漂移 hard negative |
| 标签权重 | gold 权重大于 silver；soft 只在 gold baseline 稳定后加入 |
| 划分方式 | 除 event split 外，必须加入 line split 和采样率分层指标 |

## 6. 推荐实施路线

1. 冻结当前模型，仅作为诊断 baseline，不建议导出部署。
2. 从当前最差样本和跨相冲突样本中抽取人工复核集，先做 200 个事件级 gold 样本；理想目标是 400 到 600 个事件。
3. 重写 dataset v2 生成逻辑：固定物理搜索窗、事件级标签、三相一致性、双端物理反校验。
4. 建立 candidate proposal recall 指标，先确保真实波头在 Top-K 候选中召回率达到 99% 左右。
5. 训练候选判别/排序模型，优先解决“选错波头”问题，再优化 offset 精度。
6. 评估时同时报告 event split、line split、625 kHz / 1.25 MHz 分层、线路分层、端点分层和波形类型分层。
7. 最后用已知故障点或可信双端样本验证距离误差，而不是只看相级伪标签 MAE。

## 7. 下一版模型的验收门槛

建议用以下门槛作为是否继续推进的标准：

| 指标 | 建议门槛 | 目的 |
|---|---:|---|
| candidate recall@K on gold | ≥ 99% | 确保真实波头没有被候选生成漏掉 |
| Acc@4 samples on gold | ≥ 90%，目标 ≥ 95% | 验证细粒度波头定位 |
| P95 error on gold | ≤ 8 unified samples | 控制工程尾部风险 |
| catastrophic error > 64 samples | < 1% | 防止跳到反射波或后续大脉冲 |
| line split 性能 | 不显著劣于 event split | 验证未知线路泛化 |
| 双端测距距离误差 | 优于现有信号处理基线 | 证明最终工程收益 |
| 物理边界违规率 | 接近 0 | 避免距离落到线路外或端点方向错误 |

## 8. 知识梳理

### 8.1 关键原理

行波故障定位依赖故障瞬间产生的暂态高频波头。双端法通常利用两端首波到达时间差计算故障位置，核心是首波时间拾取必须准确。一旦把反射波、后续强脉冲或噪声尖峰误判为首波，距离误差会被传播速度放大到公里级甚至百公里级。

### 8.2 数据流

当前数据流是：

```text
.all/.vall 原始录波
-> 自动波头标注器生成 hard/soft/review
-> 根据标签做 common crop
-> HDF5: [event, phase, 8192]
-> coarse-to-fine CNN 训练
-> 在 hard pseudo label 上评估 MAE / Acc@k
```

关键问题出现在 `自动标注器生成标签` 和 `根据标签裁剪` 两步。前者导致监督目标不可靠，后者导致训练输入带有不可部署的位置先验。

### 8.3 算法逻辑

当前 CNN 的粗细定位逻辑可以理解为：

1. 把 8192 点窗口划成 256 个粗 bin。
2. 模型预测每个 bin 的波头概率。
3. 取概率最大的 bin。
4. 在该 bin 内回归 offset。

这个结构在“目标唯一且标签可靠”时有效。但行波数据经常存在多个候选波头，工程语义要求选“最早可信首波”，不是选“最像标签或幅值最大的位置”。因此需要候选排序或 hazard-style first-event 建模。

### 8.4 工程边界

当前模型只能说明：在当前伪标签和当前裁剪窗口定义下，CNN 有一定拟合能力。它不能说明：

1. 可以从原始 `.all` 文件中独立找到真实波头。
2. 可以在未见过线路上稳定泛化。
3. 可以直接提高双端故障测距精度。
4. 可以替代人工复核或物理一致性检查。

### 8.5 风险点

最高风险是用错误 hard label 继续训练，模型会学习自动标注器的偏差，并在部署时把这种偏差放大。第二个风险是用 event split 指标替代未知线路泛化能力。第三个风险是只优化平均误差，而忽略 P95 和灾难性跳峰错误。

### 8.6 验证方式

下一阶段验证应优先回答三个问题：

1. 真实首波是否在候选 Top-K 中。
2. 模型是否能把真实首波排在后续反射波和强脉冲之前。
3. 选出的双端波头是否能产生物理合理、距离误差更小的故障位置。

只有这三点成立，CNN 才真正对故障测距有工程价值。

## 9. 当前报告判定

按照数据验证标准，本次分析结论为：`Needs revision`。

不是 notebook 运行失败，而是当前数据集定义、伪标签质量和评价方式不足以支撑“模型可用”或“继续调参能解决问题”的结论。下一步应先修正数据和任务定义，再进行模型结构优化。
